01 · 배경과 문제의식

왜 지금, 시공성 중심의 AI인가.

대형 건설 프로젝트의 공정 지연·원가 초과·재시공은 대부분 설계 단계에서 시공 현실을 반영하지 못해 발생한다. AI는 그간 '설계 최적화'에 머물렀지만, 실제 현장 리스크의 상당수는 설계 이후가 아니라 설계 시점에 이미 결정된다.

기존 AI 설계 지원 도구의 한계
  • 설계 파라미터 최적화에만 집중, 시공 단계 연계 부족
  • 시공 간섭·반입 동선·장비 크레인 여건을 사후에야 검토
  • 규제·안전기준을 수동 체크리스트로 검토해 누락 발생
  • 설계 변경 시 시공 공기·비용 영향이 정량화되지 않음
  • 현장 작업자가 이해 가능한 지침서로 자동 연결되지 않음
시공성 중심 AI Engineer의 접근
  • 설계와 동시에 시공 시나리오를 자율 시뮬레이션
  • 간섭·반입·장비 배치를 수천 경로로 탐색해 최적해 도출
  • 규제·표준 준수 여부를 자동 점수화 후 피드백 루프 구동
  • 설계 변경이 공기·비용에 미치는 영향 실시간 분석
  • 결과를 현장 언어 지침서·3D 다이어그램으로 자동 생성

MODULE 01 · CO-REASONING
01
CONSTRUCTOR–ENGINEER CO-REASONING

시공 중심의 아이디어 제안

두 개의 전문 에이전트 — 시공 전문가(Constructor)와 설계 엔지니어(Engineer) — 가 상호 검증하며 가설을 정교화한다. Medical AI Scientist의 'Clinician–Engineer Co-reasoning'을 건설 도메인에 이식한 구조.
1.1

시공성 기반 공동 추론

두 에이전트가 서로의 결론을 교차 검증한다. Constructor는 설계안이 '현장에서 실제로 가능한가'를, Engineer는 시공 제안이 '구조·역학적으로 안전한가'를 되묻는다.

1.2

지식 기반 가설 생성

과거 프로젝트의 시공 실패 사례, 모듈화(Modularization) 설계 가이드라인, 자재 수급 리드타임, 주요 장비 가용성 데이터를 종합해 설계 시점에 간섭·병목을 예측한다.

1.3

문헌 및 사례 검색

Semantic Scholar API와 사내 Knowledge Graph를 활용해 최신 공법·로봇 시공 기술·모듈러 사례를 검색하고 현 설계안에의 적용 가능성·신규성·타당성을 검증한다.

1.4

출력

다음 모듈(Agentic Tree Search)로 전달되는 시공 가설 세트. 각 가설은 기대 효과·예상 리스크·전제 조건을 명시한 구조화 데이터로 기술된다.

MODULE 02 · AGENTIC TREE SEARCH
02
SIMULATION · SELF-CORRECTION

시공 시뮬레이션·최적화 실행

대형 기기 반입 경로·양중 계획·공정 순서 등 수천 가지 시나리오를 자율적으로 생성·시뮬레이션하고, 오류가 발생하면 '버그'로 인식해 스스로 수정한다. The AI Scientist의 Agentic Tree Search 기술이 시공 경로·일정 최적화에 적용된다.
2.1

에이전트 기반 트리 탐색

대형 기기 반입 경로를 최적화하기 위해 수천 가지 시나리오를 자율 생성·시뮬레이션하고, 가장 짧은 시간에 안전하게 설치 가능한 경로를 찾아낸다.

2.2

자기 교정 시뮬레이션

실행기(Executor)는 물리적 충돌·장비 용량 초과 등의 오류를 '버그'로 인식하고, 코드·파라미터를 자율 수정(Debugging)해 실행 가능한 시공 계획을 재도출한다.

2.3

도메인 특화 툴박스

원자력 안전 규제, 중량물 양중 제약, 크레인 선회 반경, 기상 조건 등 도메인 제약이 반영된 시뮬레이터를 통합해, 설계 변경이 공기·비용에 미치는 영향을 실시간 분석한다.

2.4

출력

최적 시공 시나리오, 대안 시나리오 Top-N, 그리고 각 시나리오의 예상 공기·비용·리스크 지표가 다음 모듈(Reviewer)로 전달된다.

3.1

시공성 평가 점수

설계안을 여러 축 — 시공 편의성, 자재 접근성, 작업자 안전성, 간섭 회피, Soundness, Contribution — 에 따라 정량 점수화한다.

3.2

규제·표준 준수 검토

The AI Scientist의 'Ethics Reviewer'가 윤리 규정을 체크하듯, 원자력 안전기준(Regulatory Guides)·시공 표준사양·KOSHA 안전지침 준수 여부를 자동 검토한다.

3.3

피드백 루프

검토 결과 시공성이 낮은 항목은 MODULE 01(Co-reasoning)로 되먹임되어 설계 수정안이 다시 도출되도록 하는 폐루프 반복 구조를 형성한다.

3.4

출력

검토 보고서, 취약 항목 리스트, 재설계 권고안. 경계값을 통과한 최종 안이 MODULE 04(Composer)로 전달된다.

MODULE 04 · MANUSCRIPT COMPOSER
04
EVIDENCE-BASED DOCUMENTATION

시공 지침서·보고서 자동 생성

이전 모듈의 시뮬레이션 데이터와 최적화 결과를 바탕으로, 현장 작업자가 이해할 수 있는 시공 기술 지침서(Method Statement)·설계 변경 보고서·단계별 3D 다이어그램을 자동 작성한다.
시공 기술 지침서 · M/S-0347
METHOD STATEMENT · AUTO-GENERATED · REV.02
§ 1. SCOPE 대형 RV 반입·설치 작업 (Ø 6.3m × L 12m) // 자동 인용: 설계도면 DWG-A-2301, 시뮬레이션 SIM-4417 § 2. SEQUENCE 2.1 크레인 배치 C-03 위치 — 선회반경 28.4m 2.2 반입 경로 ROUTE-B-02 — 간섭 0 건 2.3 양중 작업 — 예상 소요 4h 20m § 3. SAFETY KOSHA 고용노동부 고시 준수 · 작업자 경로 분리 // 자동 첨부: 3D 다이어그램 DIAG-STEP-01 ~ 07
4.1

증거 기반 보고서 작성

시뮬레이션 로그, 점수 데이터, 재설계 히스토리를 근거로 삼아 판단 근거가 역추적 가능한 지침서·보고서를 자동 생성한다.

4.2

시각화 자료 통합

시뮬레이션 결과를 기반으로 시공 단계별 3D 다이어그램을 자동 생성하고, 간섭 지점·위험 구역을 주석 처리해 현장 의사소통 효율을 높인다.

4.3

현장 언어 최적화

공학 용어와 현장 언어를 병기하고, 경영층 보고용·실무자용·협력사 배포용 등 독자별 버전을 자동으로 분기 생성한다.

4.4

출력

Method Statement, 설계 변경 보고서, 단계별 3D 다이어그램, 설명 영상 스크립트, 경영층 1-Pager 요약 — 모든 산출물이 즉시 배포 가능한 수준으로 완성된다.

04 · 통합 워크플로우

네 개의 모듈은 하나의 폐루프로 순환한다.

초기 설계안이 입력되면 네 모듈은 자율적으로 반복 루프를 돌며 검토와 재설계를 거친다. 최종적으로 시공성 기준을 통과한 설계안만 현장으로 전달된다.

핵심은 검토 실패가 곧 재설계 트리거라는 점이다. 기존 인간 중심 검토가 "지적 → 회의 → 수정"의 긴 사이클이었다면, 본 프레임워크는 이를 분 단위 자율 반복으로 전환해 설계 완성도를 수렴시킨다.

05 · 기대 효과 및 결론

설계가 끝나는 순간, 시공이 시작된다.

본 프레임워크의 도입은 설계 품질·시공 품질·현장 안전의 세 축에서 정량적 성과를 만든다. 아래는 도입 후 예상되는 핵심 지표 개선 범위다.

−25%
REWORK · 재시공 감소

설계 기인 재시공 비용 절감

시공 간섭·병목을 설계 단계에서 선제 제거함으로써, 현장에서 발생하던 재시공과 설계 변경을 구조적으로 줄인다.

−18%
SCHEDULE · 공기 단축

핵심 공정 공기 단축

수천 시나리오 자율 시뮬레이션으로 최적 시공 순서와 반입 경로를 확보, 대형 기기 양중 등 핵심 공정의 리드타임을 단축한다.

+40%
REVIEW SPEED · 검토 속도

시공성 검토 사이클 가속

인간 중심 회의체 검토를 자율 루프로 전환해, 수 주 단위였던 설계–검토 사이클을 분 단위로 단축하고 반복 횟수를 늘린다.

시공성을 설계 이후의 점검 항목이 아니라,
설계의 출발선에서부터 작동하는 지능으로 끌어올린다.
